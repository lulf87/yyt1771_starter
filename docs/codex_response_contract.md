# Codex 输出约束

每次我给出工单后，你发给 Codex 的最后一段都要附上下面这段要求。

## 请 Codex 按这个格式输出

1. **变更摘要**
   - 这次做了什么
   - 没做什么

2. **最终目录树**
   - 只展示有变化的部分

3. **文件清单**
   - 新增了哪些文件
   - 修改了哪些文件
   - 移动/重命名了哪些文件

4. **关键实现说明**
   - 说明如何满足本工单的约束
   - 说明是否遵守 `docs/architecture_lock.md`

5. **命令与结果**
   - 列出实际运行的命令
   - 粘贴原始终端结果
   - 至少包含测试命令与结果

6. **风险和未决项**
   - 有哪些地方只是占位
   - 哪些点需要我继续确认

7. **把下面内容一并返回给我**
   - 完整 diff，或所有变更文件的完整内容
   - 测试输出全文
   - 若失败，失败原因与最小修复建议

## 你回传给我时，优先给我这三样

1. Codex 的完整回答
2. 变更后的目录树
3. 所有变更文件内容或 patch/diff

如果内容太长，至少保证：
- `docs/` 下新文件全文
- 所有新增 Python 文件全文
- 所有被修改测试文件全文
