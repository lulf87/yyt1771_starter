# Workspace 计算视图强化说明 v1（Task-019 对应）

## 目标
在不进入“手动调整”阶段的前提下，把当前 `/workspace/{session_id}` 的计算展示做得更清楚、更有层次。

本轮聚焦：
- 曲线阅读性
- 点/帧联动
- 右侧信息卡片的可读性
- 计算结果表达清晰化

不做：
- 参数调整
- ROI 编辑
- 手动点位拖拽
- live 实时接入
- 复杂图表库引入

---

## 一、当前问题

当前工作台已经有：
- 曲线
- 关键帧
- Af95
- Summary / Detail 摘要
- 阶段 stepper

但在“计算结果阅读”上还有几个问题：
1. 曲线的坐标语义不够清楚
2. 选中某个点/帧后的信息反馈还不够集中
3. 右侧摘要虽然已分层，但还缺少“当前选中分析对象”的信息区
4. Af95 的表达还偏轻，不够像“当前核心结果”

---

## 二、本轮强化方向

### 1. 曲线区更像“分析主图”
曲线图应成为 workspace 中央区域的主焦点。

建议增强：
- 明确 X 轴：Temperature (°C)
- 明确 Y 轴：Shape Metric / metric_raw
- Af95 标记更明显
- 当前选中点高亮更明显
- 空态提示更清楚

不要求做复杂坐标系统，只需最小标签和辅助线。

### 2. 点 / 帧联动更清楚
最小联动建议：
- 点击关键帧卡片 -> 高亮曲线对应点
- 点击曲线点 -> 高亮对应关键帧卡片
- 右侧显示“当前选中点/帧”的详细信息

### 3. 右侧加入“Active Selection”卡片
建议新增一张卡片，显示当前选中对象的细节：

字段建议：
- label（如 first/middle/last）
- timestamp_ms
- celsius
- metric_raw
- metric_norm（若存在）
- feature_point_px
- quality（若存在）

### 4. Af95 强化表达
建议在右侧 `Session Summary` 中提升 Af95 的视觉优先级：
- 更大的数字
- 更明确的单位
- 若无值，显示 `N/A`

---

## 三、页面布局建议（在当前结构上增强）

### 顶部状态栏
保持不变，仅微调文字层级。

### 左侧 stepper
保持当前阶段状态展示，不扩展功能。

### 中央主区
建议拆成两块：

#### A. Curve Panel
包含：
- 曲线标题
- 轴标签
- SVG 主图
- 当前点提示区（可在图下方）

#### B. Key Frame Panel
包含：
- 关键帧标题
- key frame 卡片网格
- 当前选中帧高亮

### 右侧 side panel
建议整理成 5 张卡片：
1. Workflow Stage
2. Session Summary
3. Active Selection
4. Detail Snapshot
5. Quick Actions

---

## 四、交互规则

### 页面加载后
- 拉取 summary
- 拉取 detail
- 默认选中：
  - 若存在 key_frames，默认选中 middle；否则选中第一帧
  - 曲线同步高亮对应点

### 点击关键帧
- 高亮关键帧卡片
- 高亮曲线点
- 更新 Active Selection

### 点击曲线点
- 高亮曲线点
- 若能匹配到 key_frame，则高亮对应卡片
- 更新 Active Selection

### detail 缺失
- 保持页面结构可见
- 曲线区显示空态
- 关键帧区显示空态
- Active Selection 显示无可选对象

---

## 五、视觉要求

继续沿用当前深色工业风：
- 深色背景
- 卡片式分区
- Af95 用更亮的强调色
- 选中帧/选中点有明显边框或 glow
- 但不要做重动画

---

## 六、本轮不做的事
- 不做参数调整
- 不做 ROI 手动编辑
- 不做自动/手动结果并排对比
- 不做 live 采集工作台
- 不做导出面板扩展

---

## 七、验收标准
用户进入 workspace 后，应能更容易回答这几个问题：
1. 当前结果的 Af95 是多少？
2. 当前曲线表达的是什么？
3. 当前选中的点/关键帧是哪一个？
4. 当前关键帧和曲线是否能对应起来？
