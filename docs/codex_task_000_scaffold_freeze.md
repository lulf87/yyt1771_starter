# Task-000：目录结构冻结与对齐

请先阅读：

- `docs/architecture_lock.md`
- `docs/module_map.md`
- 当前项目目录树

## 任务目标

把当前项目文件夹结构对齐到 `docs/architecture_lock.md` 规定的标准形态，先把**层级、命名、测试布局、占位文件**统一好。

注意：

这次任务不是实现业务功能，而是**先冻结结构**，确保后面所有工单都在同一约束下推进。

---

## 范围

### 允许你做的事

1. 新增缺失的目录与占位文件。
2. 为模块补充 `README.md` 或简单职责注释。
3. 调整测试目录，使 `tests/` 镜像业务目录。
4. 如有必要，移动现有测试文件到新的子目录。
5. 如有必要，修正导入路径，保证测试还能跑。
6. 保留已有已实现代码，不要无故重写。

### 不允许你做的事

1. 不要实现真实业务算法。
2. 不要接入真实相机、PLC、温控设备。
3. 不要新增任何一级业务模块。
4. 不要发明 `utils/`、`common/`、`shared/` 这类目录。
5. 不要修改已经存在的数据模型语义。

---

至少完成以下对齐：

1. `docs/` 下存在：
   - `architecture_lock.md`
